How Properties Work

Nearly all MathML templates have display properties that can be altered. In WebEQ Editor, you can edit display properties by selecting the template you are interested in then using the Properties Menu to open the appropriate properties dialog box.

Not all properties apply to every template. Depending on what you have selected, some of the Properties Menu entries may be grayed out, and within a particular properties dialog, some of the fields may be inactive.

Since selecting a template and opening a properties dialog is a common operation, WebEQ Editor gives you a way to quickly do this in one step. Double-clicking on a template listed in the MathML ancestry will open the most commonly used properties dialog for that template.

Special Values

Many MathML properties can be set to a wide range of values. In the property dialogs, these values are just fill-in areas, where you can type any value. See Size and Color Formats and the help pages for the individual property dialogs for details about the specific values accepted by each of these properties.

Two more special values that can be set for a number of properties are automatic and inherited. The precise meaning of automatic varies from property to property. However, in general, it indicates that the renderer should automatically choose property values according to standard typesetting rules.

The value inherited means that a property should take its value from an enclosing template. In cases where no enclosing template provides a value, renderers will usually try to take the value from the surrounding HTML, or choose a reasonable default.

Style Change Templates

Tasks such as setting a font color for a subexpression arise frequently when authoring equations. However, if the subexpression is something such as a fraction, for example, you can't directly set the font color, since font color isn't one of the fraction templates properties. While selecting every character contained in the fraction and changing its color property might be acceptable in a very small example, it is not practical in most cases.

To address this problem, MathML has a style change template. Any MathML property can be set for a style change template, and it then applies to all templates nested within it as appropriate. Thus, in the example of the fraction, a good solution is to first put the fraction in a style change, and edit the font color there.

You can insert a style change template from the Insert Layout menu.
