Layout Properties Dialog

MathML templates that only contain other templates, and not character data, are called layout templates. The Layout Properties Dialog allows you set set a handful of miscellaneous properties that apply to one or more layout templates.

Display style

There are two typographic styles for typesetting equations called "display style" and "inline style". In general, display style is used for equations that are centered and set off from the surrounding text, and inline style is used for equations mixed in with the text of a paragraph.

The main difference is that inline style tries to minimize the vertical space used by an equation, so inter-line spacing is effected less. Specifically, in display style, larger symbols are used for summations, integrals, etc. and limits are typeset over and under the operators instead of in the script positions. Fractions and scripts are also set in larger type faces, and with more vertical spacing.

The display style property only applies to the style change template, and by default, the display style property is 'false'. However, WebEQ Editor will automatically insert a style change template wrapper if you try to set the display style property for some other layout template.

In general, it is not a good idea to have only part of an equation in display style, so the standard way of using this property is to insert a style change template just inside the top-level <math> element, and set display style to 'true' on that. That has the effect of setting display style to true for all templates nested inside it.

Script level

Traditional mathematical typesetting employs at most three different font sizes in any given equation -- a base font size, a slightly smaller size for scripts, and a much smaller size for scripts of scripts. All other notation in an equation is usually typeset using one of these sizes as appropriate, depending on the level of nesting of templates.

In MathML, which font size a particular template will use is determined by its script level. The base font corresponds to script level 0, script size is level 1, and "script script" size is level 2. By default, the script level property is automatic and standard typesetting algorithms determine the script level of any given template. However, in some situations, you may want to explicitly set the script level to achieve a certain visual effect.

The script level property only applies to the style change template. However, WebEQ Editor will automatically insert a style change template wrapper if you try to set the script level property for some other layout template.

Foreground and Background color

The foreground color and background color properties set the foreground and background colors for layout templates, just as the background color property on the font properties dialog sets the background color for token templates that contain character data. The default values for these properties are inherited and can be set to any color value.

The background color property only applies to the style change template. However, WebEQ Editor will automatically insert a style change template wrapper if you try to set the background color property for some other layout template.

Fraction line thickness

The fraction line thickness property determines the thickness for the horizontal bar in a fraction template. It only applies to fraction templates. It can be set to any positive size value, any positive integer without units (interpreted as number of pixels), or one of the keywords 'thin', 'medium', 'thick'

Treat underscript/overscript as accent

When typesetting overscripts and underscripts, the usual practice is to use a smaller type face. However, when typesetting mathematical accents, such as a tilde or a bar, as an overscript, you typically want the font sizes to match. By setting these properties to 'true' you can instruct MathML renderers not to decrease the font size for a template being typeset and an under- or overscript.

The default value is automatic. These attributes only apply to overscript and underscript templates respectively.

See also the treat as accent operator property, which has the same effect when set on an <mo> template in the script position.
