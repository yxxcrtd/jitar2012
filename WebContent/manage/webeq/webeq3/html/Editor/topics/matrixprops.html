Table Properties

There are three property dialogs for setting table properties, one for an entire table, one for a table row and one for a table cell.

Matrix Properties Dialog

The MathML matrix template is quite complicated, with a dozen properties which customize its appearance.

Properties Taking Lists of Values

One important thing to understand is that many matrix properties actually take lists of values. The values in the list apply in sequence to the successive rows or columns in the matrix.

For example, specifying row alignment for a 3x3 matrix as top bottom center' indicates the first row should be top aligned, the middle row should be bottom aligned, and the last row should be centered vertically.

The values in all matrix property lists are separated by spaces. Also, if there are fewer entries in the list than rows or columns, the last value will be repeated for all the remaining rows or columns. Thus, specifying column alignment as just 'center' for a matrix indicates all columns should be centered.

Alignment Properties

Align

This property controls the alignment of the entire matrix relative to the rest of the equation. The possible values are:

'axis'
This aligns the center of the matrix with the 'axis' of the equation. The axis of the equation is basically the horizontal line through the minus sign.
'top'
This value aligns the top of the matrix with the baseline of the equation.
'center'
This value places the center of the matrix on the baseline of the equation.
'bottom'
This aligns the bottom of the matrix with the baseline of the equation.
'baseline'
This has the same effect as 'center'.

Row Align

The row align property controls the vertical alignment of table cells within a row. Row align accepts a list of alignment values which apply to successive rows. The individual values in the property list must be chosen from the keywords 'axis', 'top', 'center', 'bottom', and 'baseline'. These keywords have the following meanings.

'axis'
This aligns the natural axes of all cells in a row. The axis of a cell is the horizontal line through the minus sign.
'top'
This value aligns the tops of all cells in a row.
'center'
This value aligns the center lines of all cells in a row.
'bottom'
This value aligns the bottoms of all cells in a row.
'baseline'
This value aligns the natural baselines of all cells in a row. This is the default value.

Column Align

The column align property controls the horizontal alignment of table cells within a column. Column align accepts a list of alignment values which apply to successive columns. The individual values in the property list must be chosen from the keywords 'left', 'center', and 'right'. These keywords have the following meanings.

'left'
This value aligns the left edge of all cells in a column.
'center'
This value centers cells in a column.
'right'
This value aligns the right edges of cells in a column.

Spacing and Separator Properties

Row/Column Spacing

The row spacing and column spacing properties both accept lists of values. The values in the list determine the amount of space between successive rows and columns respectively. Each value in the list needs to be a size value.

Row/Column Lines

The row lines and column lines properties accept lists of values which indicate what kinds of separator lines (if any) should be drawn between successive rows and columns respectively. Each of the values in the list must be one of 'none', 'solid', and 'dashed'.

Frame

The frame property determines whether a rectangular box or "frame" will be drawn around the matrix. The possible values are 'none', 'solid', 'dashed'

Frame Spacing

The frame spacing is a list of up to two size values. The first value determines the space between the sides of the matrix and the frame, and the second value determines the space between the top and bottom of the matrix and the frame. Note that frame spacing is still applied even when the frame property is set to 'none', in which case the frame spacing has the effect of inserting padding between the matrix and the surrounding content

Equal Rows/Equal Columns

These properties determine whether the heights of all rows and the widths of all columns in a matrix are equal. By default, rows and columns are sized to match the tallest and widest of their entries respectively. However, if the equal rows or equal columns properties are set, then the heights and widths are determined by the tallest and widest entries in the entire matrix instead.
