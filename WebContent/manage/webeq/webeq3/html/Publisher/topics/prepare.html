Preparing the Source Document

In order for WebEQ Publisher to automatically detect equation markup to be converted into images or applet code, the equation markup must be set off from the rest of the HTML input by equation delimiters.

The Publisher recognizes two kinds of equation delimiters. WebTeX equations in the HTML source file can be delimited with either of the two standard LaTeX delimiters for mathematical expressions:

$ ... $ for in-line equations
\[ ... \] for displayed equations

Equations surrounded by the LaTeX display equation delimiters will appear centered and set off by vertical space from the surrounding text in the output document.

MathML markup should always be delimited by the standard top-level MathML <math> ... </math> tags. When using MathML, displayed equations must be explicitly marked up by the author using <P> and <CENTER> tags.

By default, WebEQ Publisher will automatically process all equations it finds, whether they are encoded as WebTeX or MathML. If you wish to convert only one kind of markup, you can specify this in the "Math Encoding" field in the Source Document section of the main Publisher window.

Special Characters

Dollar signs require special attention when you use them in a Publisher source file, since ordinarily the Publisher will interpret a dollar sign as beginning a WebTeX equation. To get a $ symbol to show up in the HTML output file, enter \$ in the input file.

Comments

The Publisher will not process any math markup embedded in an HTML comment,

<!--  .... -->

Preformatted text inside

<PRE>  ....  </PRE>

tags will also be ignored by the Publisher, so that you can include MathML or WebTeX source code in your page without having to worry that it will be turned into an applet.
