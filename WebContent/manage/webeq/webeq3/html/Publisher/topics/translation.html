Translation Options

You can open the Translation Options panel from the Options menu on the WebEQ Publisher main screen. These options primarily effect Viewer Control code and Strict MathML generated by the the Publisher.

Send errors output to HTML
Check this box to place any processing error messages generated by the Publisher as HTML in the output file. This box is checked by default.
Send error output to file:
Checking this box sends any error messages to the file you specify.
Include namespace declarations
This pulldown menu selects what namespace declarations, if any, WebEQ Publisher should insert in output document headers. By default, the Publisher will generate no declarations, and insert no namespace prefixes into MathML tags. If you choose to generate declarations, then the "Use namespace prefix" field becomes active, allowing you to set what string to use for the namespace prefix. The namespace declarations that WebEQ Publisher can generate are:

  • Mozilla namespace declarations - generates declarations identifying the MathML markup, but does not generate declarations for any specific rendering behavior. Documents prepared this way are viewable only in browsers with full native MathML support such as Mozilla or Amaya.
  • Viewer Control namespace declarations - generates declarations for using the Viewer Control as an Internet Explorer rendering behavior.
  • MathPlayer namespace declarations - generates declarations for using Design Science's MathPlayer as an Internet Explorer rendering behavior.
  • Techexplorer namespace declarations - generates declarations for using IBM's Techexplorer plug-in as an Internet Explorer rendering behavior.
  • Techexplorer namespace declarations - generates declarations for using IBM's Techexplorer plug-in as an Internet Explorer rendering behavior.
  • Math Stylesheet namespace declarations -- generates declarations for using the W3C XSL Math Stylesheet. The stylesheet attempts to detect MathML rendering options available in the client Web browser, and dynamically regenerates the page to take advantage of the best one.

Use namespace prefix
This field specifies the MathML namespace prefix label to use. It is arbitrary, but must be unique within the output document. However, for Techexplorer, only mml is allowed. This field is only active when namespace declarations of some kind are being generated. By default, this option is set to use "m:" as the namespace prefix.
Export symbols as:
Use the drop-down menu to select one of three choices for this option: "MathML entity names," "Numerical references," or "Unicode characters." These options determine how WebEQ Publisher will encode symbol characters in the output document. Many symbol characters have MathML names, such as &alpha;. Alternatively, this character also can be specified by its hexadecimal numerical Unicode value as &#x03b1;. Finally, a binary representation of the raw Unicode character "alpha" can be inserted in the output document.

At the time of this writing, no clear concensus has emerged in the MathML community as to which format to use. Most MathML applications understand all three formats, but some don't. Until there is a clear choice, you will have to experiment to see what works best for you and your readers.

Note: When using a namespace declaration with Mozilla or Math Stylesheets, the "MathML Entity Names" option is not available.

Translate WebTeX to MathML in applets
When this option is checked, the Publisher will convert any WebTeX it encounters in the source document to MathML. This is only relevant when the output equation format is MathML or Viewer Control applet code.
Local macro file for WebTeX
Specifies the location of a WebTeX macro file for use during processing.
