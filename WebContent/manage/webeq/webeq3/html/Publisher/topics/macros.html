Using WebTeX Macros

A useful feature of WebEQ Publisher is its ability to process document-wide macro definitions; if you are a WebTeX author, you can introduce shorthand commands for your commonly used notations.

WebTeX macro definitions can either be specified once at the beginning of a source document, or in a separate file. The Publisher facilitates both kinds of macro definitions.

Specifying Macros in the Document Header

When WebEQ Publisher runs, it looks for WebTeX macro definitions in a special comment block in the HTML source file header. The syntax of this comment block is illustrated below:

<html>
<head>
<title>An Example&lt;/title>

<!-- webeq
\define{\gump}{x^2}
-->

</head>
<body>

This is a test $\gump = 9$.

</body>
</html>

Note: The comment block must exactly match this format, including whitespace characters. Provided the "Translate WebTeX to MathML in applets" box is not checked, the Publisher will generate an HTML output file like this:

&lt;html>
&lt;head>
&lt;title>An Example&lt;/title>
&lt;/head>
&lt;body>

This is a test

<b>&lt;applet code="webeq.ViewerControl" height="30" width="54">
&lt;param name="eq" value="x^2 = 9">
&lt;applet></b>

&lt;/body>
&lt;/html>

Specifying Macros in File

If you wish to use a large collection of macros, or if you wish to share macros between a number of HTML files, it will probably be more convenient to specify macro definitions in an external file. A macro file can be used in conjunction with macro definitions given in the document header, for example, to extend a base set of macros in an external file.

You could do the previous example using a macro file as follows. Create a macro file named macros.txt containing the single line

\define{\gump}{x^2}

Then, you would create essentially the same source document, but without the macro definition in the document header:

<html>
<head>
<title>An Example</title>
</head>
<body>

This is a test $\gump = 9$.

</body>
</html>

In the Translation Options dialog, you would then enter the location of macros.txt in the "Local macro file for WebTeX" field. The output would be exactly the same as in the case where the macros were defined in the source document header.
